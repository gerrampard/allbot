# AllBot 配置指南

## 1. 配置文件说明

AllBot 使用 TOML 格式配置文件。主配置文件位于项目根目录的 `main_config.toml`，建议基于 `main_config.template.toml` 复制后修改。

插件与适配器各自拥有独立配置文件，位于 `plugins/<name>/config.toml` 与 `adapter/<name>/config.toml`。

## 2. 配置结构总览

`main_config.toml` 的主要模块如下：

1. 协议设置（`[Protocol]`）
2. 框架设置（`[Framework]`）
3. WechatAPI 服务配置（`[WechatAPIServer]`）
4. Redis 设置（兼容顶层 `redis-*` 旧写法）
5. WebSocket 设置（兼容顶层 `enable-websocket` / `ws-url` 旧写法）
6. RabbitMQ 设置（兼容顶层 `enable-rabbitmq` 等旧写法）
7. 管理后台配置（`[Admin]`）
8. 日志与性能监控（`[Logging]`、`[Performance]`）
9. 核心功能（`[XYBot]`）
10. 自动重启（`[AutoRestart]`）
11. 消息过滤（`ignore-mode`、`whitelist`、`blacklist`）
12. 通知系统（`[Notification]`）

## 3. 关键配置说明

### 3.1 协议设置

```toml
[Protocol]
version = "869"  # 869/ipad/pad/mac/ipad2/car/win
```

### 3.2 框架设置

```toml
[Framework]
type = "wechat"
```

### 3.3 WechatAPI 服务配置

```toml
[WechatAPIServer]
host = "127.0.0.1"
port = 9000
mode = "debug"  # release/debug
admin-key = ""  # 869 管理密钥，部署时填写
```

### 3.4 Redis 配置

```toml
redis-host = "127.0.0.1"
redis-port = 6379
redis-password = ""
redis-db = 0
```

Redis 用于消息队列与状态存储，适配器和核心处理流程均依赖该配置。当前加载器兼容模板里的顶层写法，也兼容写进 `[WechatAPIServer]` 段内。

### 3.5 WebSocket 设置

```toml
enable-websocket = true
ws-url = "ws://127.0.0.1:9000/ws/GetSyncMsg"  # 869 主链路，程序会自动追加 ?key=
login-qrcode-proxy = ""  # 869 拉码代理，可选，例如 socks5://127.0.0.1:1080
```

### 3.6 RabbitMQ 设置

```toml
enable-rabbitmq = true
rabbitmq-host = "127.0.0.1"
rabbitmq-port = 5672
rabbitmq-user = "guest"
rabbitmq-password = "guest"
rabbitmq-queue = "example-queue"
```

以上 WebSocket / RabbitMQ 字段同样兼容顶层旧写法与 `[WechatAPIServer]` 段内写法；为减少歧义，建议同一份配置只保留一种写法。

### 3.7 管理后台配置

```toml
[Admin]
enabled = true
host = "0.0.0.0"
port = 9090
username = "admin"
password = "change_me"
secret-key = "change_me_to_a_random_secret"
session-cookie-secure = false
cors-origins = ["http://127.0.0.1", "http://localhost"]
debug = true
log_level = "INFO"
```

- `password` 不能继续使用示例值，至少应替换为强密码。
- `secret-key` 是管理后台会话签名密钥，必须替换为至少 24 位随机字符串；缺失、过弱或保持默认值时，管理后台会直接拒绝启动。
- `session-cookie-secure` 控制会话 Cookie 是否仅通过 HTTPS 发送；本地 HTTP 调试可为 `false`，正式 HTTPS 部署应改为 `true`。
- `cors-origins` 是允许访问后台的前端来源白名单；开发环境可保留本机地址，生产环境应仅保留实际域名。

### 3.8 日志与性能监控

```toml
[Logging]
enable_file_log = true
enable_console_log = true
enable_json_format = false
max_log_files = 10
log_rotation = "1 day"

[Performance]
enabled = true
monitoring_interval = 30
max_history_size = 1000
cpu_alert_threshold = 80
memory_alert_threshold = 85
memory_low_threshold_mb = 500
```

### 3.9 核心设置

```toml
[XYBot]
enable-wechat-login = false
ignore-protection = false
enable-group-wakeup = false
group-wakeup-words = ["bot", "机器人"]
robot-names = ["bot"]
robot-wxids = ["wxid_xxxxxxxxxxxxx"]
github-proxy = ""
XYBotDB-url = "sqlite:///database/xybot.db"
msgDB-url = "sqlite+aiosqlite:///database/message.db"
keyvalDB-url = "sqlite+aiosqlite:///database/keyval.db"
admins = ["wxid_admin_1", "wxid_admin_2"]
disabled-plugins = []
timezone = "Asia/Shanghai"
auto-restart = false
files-cleanup-days = 1
```

`enable-wechat-login = false` 表示仅依赖适配器消息输入（如 Web 对话、QQ/TG 适配器）。

### 3.10 自动重启监控

```toml
[AutoRestart]
enabled = false
check-interval = 60
offline-threshold = 300
max-restart-attempts = 3
restart-cooldown = 1800
check-offline-trace = true
failure-count-threshold = 10
reset-threshold-multiplier = 3
```

### 3.11 消息过滤

```toml
ignore-mode = "None"  # None/Whitelist/Blacklist
whitelist = ["wxid_example_1", "1234567890@chatroom"]
blacklist = ["wxid_block_1", "333@chatroom"]
```

### 3.12 通知系统

```toml
[Notification]
enabled = false
token = ""
channel = "wechat"
template = "html"
topic = ""
heartbeatThreshold = 3

[Notification.triggers]
offline = true
reconnect = true
restart = true
error = true

[Notification.templates]
offlineTitle = "警告：微信离线通知 - {time}"
offlineContent = "..."
```

## 4. 适配器独立配置

适配器配置文件位于 `adapter/<name>/config.toml`。其中 `ocwx` 表示微信 clawbot 渠道，配置文件路径为 `adapter/ocwx/config.toml`。

### 4.1 ocwx 最小模板

```toml
[adapter]
name = "ocwx"
enabled = false
module = "adapter.ocwx"
class = "OpenClawWeixinAdapter"
replyQueue = "allbot_reply:ocwx"
replyMaxRetry = 3
replyRetryInterval = 2
logEnabled = true
logLevel = "INFO"

[ocwx]
enable = false
platform = "ocwx"

[ocwx.defaults]

[ocwx.redis]

[ocwx.accounts]
```

`[adapter]` 与 `[ocwx]` 是最小必需段；其余三个配置段只用于提示结构，可全部留空。

### 4.2 ocwx 可选配置段

- `[ocwx.defaults]`：覆盖上游接口地址与轮询/超时参数，支持 `baseUrl`、`cdnBaseUrl`、`pollTimeoutMs`、`requestTimeoutMs`、`loginCheckIntervalSec`、`qrRefreshIntervalSec`。
- `[ocwx.redis]`：覆盖专用 Redis 连接，支持 `queue`、`host`、`port`、`db`、`password`。
- `[ocwx.accounts.<slot>]`：声明账号槽位；`<slot>` 可替换成 `main`、`backup` 等任意名称，常用字段为 `enabled`、`displayName`，也可按账号覆盖 `baseUrl`、`cdnBaseUrl`、`pollTimeoutMs` 等。

### 4.3 真实默认值来源

`adapter/ocwx/config.toml` 现在是空模板，不再携带运行示例值。省略配置时，真实默认值来自 `adapter/ocwx/ocwx_adapter.py`：

- `stateDir = "resource/ocwx"`
- `qrDir = "admin/static/temp/ocwx"`
- `replyMediaDir = "admin/static/temp/ocwx/reply-media"`
- `mediaCacheDir = "admin/static/temp/ocwx/media"`
- `baseUrl = "https://ilinkai.weixin.qq.com"`
- `cdnBaseUrl = "https://novac2c.cdn.weixin.qq.com/c2c"`
- `pollTimeoutMs = 35000`
- `requestTimeoutMs = 15000`
- `loginCheckIntervalSec = 3`
- `qrRefreshIntervalSec = 30`
- Redis 优先读取 `main_config.toml` 的 `WechatAPIServer.redis-*`，再回退到 `127.0.0.1:6379/0`

### 4.4 启用 ocwx 前的最低要求

1. 至少新增一个 `[ocwx.accounts.<slot>]` 账号槽位。
2. 把 `[adapter].enabled` 与 `[ocwx].enable` 改为 `true`。
3. 如果默认 Redis 或默认 `openclaw-weixin` 地址不适合当前环境，再补充 `[ocwx.redis]` 或 `[ocwx.defaults]`。

## 5. 使用建议

- 生产环境建议设置强密码，并关闭 `debug`。
- 若开启适配器模式（`enable-wechat-login = false`），需确保 Redis 可用。
- 修改配置后建议重启服务确保生效。
